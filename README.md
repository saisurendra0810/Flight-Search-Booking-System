# Flight Search and Booking System

## Project Overview

The Flight Search and Booking System is a web-based application designed to simplify the process of searching, viewing, and booking airline tickets. The system enables users to search for flights based on source and destination locations, compare available flight options, view flight details, and complete the booking process through an interactive and user-friendly interface.

This project demonstrates the implementation of modern front-end development concepts including component-based architecture, state management, data handling, input validation, and application testing.

---

## Objectives

* Provide a simple and efficient flight search mechanism.
* Display available flight information in a clear format.
* Allow users to select and book flights online.
* Validate passenger information before booking confirmation.
* Manage dynamic application data efficiently.
* Ensure application reliability through testing.
* Deliver a responsive and user-friendly experience.

---

## Features

### Flight Search

Users can search for available flights using source and destination details.

### Flight Details Display

The system displays flight information such as:

* Airline Name
* Source Location
* Destination Location
* Departure Time
* Arrival Time
* Ticket Price

### Flight Booking

Users can select a preferred flight and proceed with booking.

### Passenger Information Management

The system collects and manages passenger details required for reservation.

### Input Validation

Passenger information is validated before confirming the booking.

### Booking Confirmation

The system generates a confirmation message after successful booking.

## Course Outcome (CO) Implementation

### CO1: Displaying Flight Details Using Functions

Flight information is displayed using reusable functions that process and present flight details in a structured format.

Flight-related data is organized using ES6 classes, while arrow functions are used for efficient data processing and operations.

Flight records are retrieved, processed, filtered, and displayed based on user search criteria.

Passenger details are validated to ensure data accuracy and prevent invalid bookings.

### CO6: Testing Application Functionality

Different modules of the system are tested to verify proper functionality and improve reliability.

## System Architecture

The project follows a layered architecture consisting of:

### User Layer

Allows users to interact with the application.

### Presentation Layer

Handles user interface components and displays information.

### Business Logic Layer

Processes flight searches, validations, and booking operations.

### Data Layer

Stores flight information and booking records.

---

## Project Flow

1. User opens the application.
2. User enters source and destination details.
3. System searches for matching flights.
4. Available flights are displayed.
5. User selects a flight.
6. Passenger information is entered.
7. System validates the entered details.
8. Booking is confirmed.
9. Confirmation details are displayed to the user.
